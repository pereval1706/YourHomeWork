# Шпаргалка по Gitу #

*git init* - команда, инициализирующая докальный репазиторий

*git version* - команда, отображающая текущую версию git.

*git diff* - команда для сравнения с последним коммитом.

*git commit -m"message"* - команда для добавления сообщения после изменения.

*git log* - команда, отображающая список коммитов.

*git branch* - команда позволяет создавать, просматривать, переименовывать и удалять ветки.

*git checkout* - команда для переключения между ветками.

# Инструкция для работы с Markdown

## Выделение текста

Чтобы выделить текст курсивом, необходимо обрамить его звездочками (*) или знаком нижнего подчеркивания (_). Например *вот так* или _вот так_

Чтобы выделить текст полужирным, необходимо обрамить его двойными звездочками (**) или двойным знаком нижнего подчеркивания. Например, **вот так** или __вот так__.

Альтернативные способы выделения текста жирным или курсивом нужнф для того, чтобы мы могли совмещать эти оба способа. Например, _текст может быть выделен курсивом и при этом **полужирным**_. 

## Списки

Чтобы добавить ненумерованные списки, необходимо пункты выделить звездочкой (*) или знаком +.Например, вот так:
* Элемент 1 
* Элемент 2
* Элемент 3
+ Элемент 4

Чтобы добавить нумерованные списки, необходимо пункты просто пронумеровать. Например, вот так:
1. Первый пункт
2. Второй пункт

## Работа с изображениями

Чтобы вставить изображение в текст, достаточно написать следующее:
![Привет, это МаксРес](maxresdefault.jpg)

## Работа с ветками

Для просмотра списка существующих веток необходимо использоватьь след команду:

   git branch

   Команда *git status* позволяет просмотреть текущее состояние, в том числе - название текущей ветки

### Добавление новой ветки

Для добавления новой ветки изпользуется следующая команда("branch_name" - название ветки):

     git branch branch_name

### Переход между ветками 

Для перехода между ветками используется следующая команда ("branch_name" - название ветки, в кот осущ переход):

     git checkout branch_name

### Удаление веток

Для удаления втеки необх исполь след команду("branch_name" - название удаляемой ветки):

    git branch -d branch_name

*Внимание!* перед удалением ветки необх удост, что данные сохранены и данная ветка более не потребуется

### Слияние веток

Для слияния двух веток применяется след команда:

     git merge branch_name

*Важные моменты при слиянии*:
* В момент слияния мы находимся в ветку, куда будут загружены изменения;
* "branch_name" - название ветки, изм кот будут отправляться

### Последствия слияния веток

При слиянии могут возникнуть ситуации, когда в основной ветке есть противоречия относительно изменений, получаемых с побочной ветки. В таких случаях 

* оставить в основной ветке ее версию данных;
* заменить данные основной ветки на данные вливаемой ветки;
* оставить оба варианта, для возможности вручную указать итоговые данные.

### Как делать Pull request

Найти в Github нужный репозиторий.

Форкнуть его (Fork), то есть скопировать репозиторий на свой аккаунт, чтобы потом иметь возможность изменить его.

Скопировать ссылку скопированного репозитория
(Git clone) ссылка скопированного адреса

Cd (change directory) название файла, который склонировали, чтобы перейти в него

__*Важно помнить!!!*__ Перед изменением склориванного репозитория, необходимо создать новую ветку.Производим все изменения только в этой ветке.

Отправляем эти изменения на свой аккаунт (push).

В окне на Github появляется возможность отправить pull request
